`inline` в C++ — это ключевое слово, которое предлагает компилятору встроить тело функции непосредственно в место её вызова, заменяя сам вызов, чтобы избежать накладных расходов на передачу управления и работу со стеком, что потенциально ускоряет программу, особенно для коротких функций, но может увеличить размер кода. Это директива, а не команда: компилятор может проигнорировать её, если считает, что функция слишком большая или есть другие причины для оптимизации. 

Обычный вызов функции:
Программа переходит к адресу функции в памяти, передает аргументы, выполняет код, а затем возвращается обратно. Это занимает время.

`inline`-вызов: 
Код самой функции "копируется" и "вставляется" в то место, где она была вызвана, как будто вы написали ее код там. 

Ускорение работы программы за счет устранения затрат на вызов функции. 

Может увеличить размер исполняемого файла (из-за дублирования кода), особенно если функция большая, что может замедлить работу программы,. 

- Для очень маленьких и часто вызываемых функций (например, геттеров/сеттеров, простых математических операций).
- Когда производительность критически важна.
- Важно: Не стоит злоупотреблять `inline`, так как современные компиляторы сами прекрасно справляются с оптимизацией, часто делая `inline` излишним или даже вредным. 

```c++
// Без inline
int add(int a, int b) {
	return a + b
}

// С inline
inline int add_inline(int a, int b) {
	return a + b;
}

int main() {
	int sum1 = add(5, 3); // Вызов функции
    int sum2 = add_inline(5, 3); // Компилятор может вставить '5 + 3' прямо сюда
	return 0;
}
```
