**NRVO (Named Return Value Optimization)** — это оптимизация компилятора в языках программирования типа C++, которая позволяет избежать ненужного копирования или перемещения локальных объектов при их возврате из функции, конструируя объект сразу в месте вызова функции, вместо того чтобы создавать временный объект, а затем его копировать или перемещать, значительно повышая производительность. 

*Без NRVO*: Функция создает локальный объект, затем копирует или перемещает его в место вызова, где он будет использоваться. Это приводит к лишним операциям (конструкторам копирования/перемещения).
*С NRVO*: Компилятор видит возвращаемое имя локальной переменной (например, `return my_object;`) и вместо создания временного объекта, он создает этот `my_object` непосредственно в памяти целевого места, куда должен быть возвращен результат.
Включено по умолчанию: Во многих современных компиляторах, включая MSVC, эта оптимизация включена по умолчанию (*флаг /Zc:nrvo*), чтобы повысить эффективность кода. 

```c++
struct MyClass {};

MyClass createObject() {
    MyClass obj; // Локальный объект
    return obj;  // Возвращаем локальный объект
}

int main() {
    MyClass result = createObject(); // Здесь NRVO сработает
    return 0;
}
```

`obj` будет сконструирован сразу как `result` в main, минуя промежуточное копирование.
