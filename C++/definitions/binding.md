*Связывание (binding)* в C++ — это процесс сопоставления вызова функции с конкретным телом (определением) этой функции. 

**1. Раннее связывание (Статическое)**
Происходит на этапе *компиляции*. Компилятор заранее знает, какую именно функцию вызвать, основываясь на типе переменной, указателя или ссылки. 

- Когда используется:
    - *Обычные (невиртуальные) функции*.
    - *Перегрузка функций* ([[overloading]]).
    - *Перегрузка операторов.*
- Плюсы: Высокая производительность, так как нет накладных расходов во время выполнения программы.
- Минус: Отсутствие гибкости — нельзя изменить поведение функции в зависимости от реального типа объекта во время работы программы. 

**2. Позднее связывание (Динамическое)**
Происходит на этапе *выполнения (runtime)*. Программа определяет, какую функцию вызвать, только в момент обращения к ней, исходя из фактического типа объекта, на который указывает ссылка или указатель. 

- Когда используется:
    - *Виртуальные функции ([[virtual]]).
    - *Указатели на функции*.
- Механизм: В C++ чаще всего реализуется через *таблицу виртуальных методов ([[vtable]])* и указатель на неё (`vptr`) внутри объекта.
- Плюсы: *Обеспечивает полиморфизм* — возможность работать с объектами разных производных классов через интерфейс базового класса.
- Минус: Незначительная потеря производительности из-за необходимости поиска адреса функции в таблице при каждом вызове. 

| Характеристика           | Раннее связывание    | Позднее связывание        |
| ------------------------ | -------------------- | ------------------------- |
| **Время определения**    | Компиляция           | Выполнение (Runtime)      |
| **Основание для выбора** | Тип указателя/ссылки | Фактический тип объекта   |
| **Ключевые слова**       | Нет (по умолчанию)   | virtual                   |
| **Скорость**             | Быстрее              | Медленнее (vtable lookup) |
