**Small String Optimization (SSO) (Оптимизация коротких строк) в C++** — это техника, позволяющая классу `std::string` (и подобным) хранить короткие строки прямо внутри объекта, а не выделять для них память в куче (heap), тем самым избегая накладных расходов на выделение/освобождение памяти и повышая производительность, особенно при частых операциях с коротким текстом. 

**Как это работает:**
1. **Встроенный буфер:** Внутри объекта `std::string` есть небольшой, фиксированный буфер (например, 15-23 байта, зависит от реализации).
2. **Хранение:** Если длина строки меньше размера этого буфера (включая нуль-терминатор `\0`), она хранится прямо там, без выделения динамической памяти.
3. **Переключение:** Если строка становится слишком длинной, память выделяется в куче, а объект `std::string` начинает указывать на этот блок памяти, как обычно. 

**Преимущества:**
- **Скорость:** Ускоряет создание, копирование и присваивание коротких строк.
- **Эффективность:** Сокращает количество выделений памяти в куче, что снижает фрагментацию и улучшает использование кэша. 

```cpp
#include <string>
#include <iostream>

int main() {
    std::string short_str = "Hello"; // Вероятно, хранится внутри объекта (SSO)
    std::string long_str = "This is a very long string that will likely go to the heap"; // Выделяется память в куче

    // Добавление символов к короткой строке может привести к её "переезду" в кучу
    short_str += " World";

    std::cout << short_str << std::endl;
    return 0;
}
```

SSO — это не часть стандарта, а распространённая _оптимизация_ в реализациях стандартных библиотек (GCC, Clang, MSVC), которая значительно улучшает работу с типичными строковыми данными
