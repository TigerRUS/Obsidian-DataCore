**RVO (Return Value Optimization)** - Оптимизация Возвращаемого Значения в C++ — это мощная техника оптимизации компилятора, которая устраняет создание временных объектов при возврате значений из функций, конструктивно создавая возвращаемый объект сразу на месте его использования, что экономит ресурсы и улучшает производительность, особенно для тяжелых объектов. Это частный случай более *общей оптимизации Copy Elision* (исключение копирования), которая гарантирована стандартом C++17. 

Без RVO:
```c++
MyClass createObject() {
    MyClass temp; // Создание временного объекта temp
    // ...
    return temp; // Копирование temp в возвращаемый объект
}

MyClass obj = createObject(); // Копирование возвращаемого объекта в obj
```
Здесь происходит создание `temp`, затем его копирование/перемещение, и, наконец, копирование в `obj`.

Использование RVO:
```c++
MyClass createObject() {
    MyClass temp; // Создание временного объекта temp
    // ...
    return temp; // Компилятор понимает, что temp будет возвращен
}

MyClass obj = createObject(); // Создается сразу как obj (без копирования)
```
Компилятор видит, что `temp` будет возвращен, и вместо создания временного объекта, который потом копируется, он сразу создаст `obj` внутри `createObject()`, а затем переместит его в `obj` вне функции, или, в случае [[NRVO]] (Named Return Value Optimization), вообще избежит копирования, создав объект сразу в месте вызова.
