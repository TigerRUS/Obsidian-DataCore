**Выравнивание памяти (Memory Alignment)** — это расположение данных в памяти по адресам, кратным размеру данных (например, `int` по адресу, кратному 4 байтам), что ускоряет доступ процессора, который работает с машинными словами; это достигается неявным добавлением "пустых" байтов (padding) в структуры и классы, но может быть изменено с помощью директив компилятора, таких как `__attribute__((aligned))` или `#pragma pack`, для оптимизации памяти или совместимости с аппаратным обеспечением. 

У каждого типа данных есть свой *коэффициент выравнивания*. Например, `char` (1 байт) может лежать по любому адресу, а `int` (4 байта) обычно должен начинаться по адресу, кратному 4.

Если вы располагаете переменные вперемешку, то компилятор вынужден вставлять *padding* (пустые байты), чтобы следующая переменная оказалась на оптимальном адресе памяти.

```cpp
struct BadStruct {
    char a;      // 1 байт
    // 3 байта padding (чтобы int встал на адрес кратный 4)
    int b;       // 4 байта
    char c;      // 1 байт
    // 7 байт padding (чтобы double встал на адрес кратный 8)
    double d;    // 8 байт
};
```
**Результат:** `sizeof(BadStruct)` будет *24 байта*, хотя полезных данных в ней всего 14 байт. Почти половина памяти тратится впустую.

Золотое правило: *сортируйте переменные от большего размера к меньшему.*
```cpp
struct GoodStruct {
    double d;    // 8 байт
    int b;       // 4 байта
    char a;      // 1 байт
    char c;      // 1 байт
    // 2 байта padding в конце (чтобы размер структуры был кратен 8)
};
```
**Результат:** `sizeof(GoodStruct)` составит **16 байт**. Мы сэкономили 8 байт на ровном месте.

Почему это критично:
1. **Экономия памяти:** В больших массивах структур разница в 8-12 байт превращается в мегабайты сэкономленной RAM.
2. **Эффективность кэша:** Чем компактнее структура, тем больше их помещается в кэш-линию процессора (L1/L2 cache). Это дает колоссальный прирост производительности.
3. **Атомарность:** На некоторых архитектурах невыровненный доступ к данным может привести к падению программы (`bus error`) или отсутствию атомарности операций.

**Как управлять выравниванием**
- **Неявное (автоматическое)**: Компилятор управляет этим сам. Переупорядочивание полей структуры (сначала большие, потом маленькие) может уменьшить padding.
- **Явное (директивы компилятора)**:
    - `__attribute__((aligned(N)))`: Для GCC/Clang (например, `__attribute__((aligned(16)))`).
    - `__declspec(align(N))`: Для MSVC (например, `__declspec(align(16))`).
    - `#pragma pack(N)`: Для установки упаковки (например, `#pragma pack(1)` для полного отсутствия padding). 

**Когда это нужно**
- **Встраивание в аппаратные регистры**: При работе с драйверами или низкоуровневым кодом.
- **Сетевые протоколы/Файловые форматы**: Когда структура должна соответствовать определенному формату.
- **Оптимизация памяти**: Если экономия памяти критична, можно использовать `#pragma pack(1)` и жертвовать производительностью.

Как проверить выравнивание
- Используйте `sizeof(MyStruct)` для общего размера.
- Используйте `alignof(MyStruct)` (C++11+) для получения размера выравнивания.
- Проверяйте адреса полей с помощью `&myStruct.field1`, `&myStruct.field2` для понимания padding.
- `static_assert` для проверки размера структуры во время компиляции.

---

Поэтому компиляторы языка программирования C (да и большинства других тоже) проводят такую оптимизацию: *между полями структуры вставляются незначащие байты для того, чтобы у всех полей было хорошее выравнивание*.

Пример: для такой структуры:
```cpp
struct
{
    char x; // 1 байт
    int y;  // 4 байта
    char z; // 1 байт
};
```

память выделяется так:
```cpp
[    ] [    ] [    ] [    ] [    ] [    ] [    ] [    ] [    ] [    ] [    ] [    ]
[  x ] { потерянные байты } [             y           ] [  z ] { потерянные байты }
```

Если сама структура будет выровнена в памяти (об этом компилятор тоже заботится), то `y` будет выровнено, и доступ к нему будет быстрым (а на некоторых платформах, напомню, вообще только в этом случае возможен).

Но в этом случае наша структура занимает больше памяти, чем если бы порядок переменных был таким:
```cpp
[    ] [    ] [    ] [    ] [    ] [    ] [    ] [    ]
[  x ] [  z ] {  потеряно } [             y           ]
```

Бóльшая структура означает бóльший расход памяти и бóльшие затраты на копирование, чтение этой структуры и тому подобное. Таким образом, *переставив поля структуры, мы можем сэкономить.*

**Выравнивание памяти** объязательно нужно для:
- для выполнения [[SIMD]] (Single Instruction Multiple Data)
- для атомарных операций ([[atomics]])
