в стандартном С++ нет типа данных строка

```c++
сhar symbol = 'c' //тип данных символ записывается в одинарных кавычках
```

```c++
сhar symbol = "с" //ошибка компиляции
```
т.к. в переменную пытаемся записать 2 символа, второй символ - это *нуль терминатор (терминирующий нуль)* `\0` 

**Важно: При использовании двойных кавычек компилятор распознаёт это как строку и вставляет в конец** `\0`

в C++ множество функций проверяют конец строки *по наличию нуля терминатора*, например `std::strlen()`

```c++
char string [] = "c" //запись корректна, так как символ и \0 помещаются в массив
```

```c++
char string [256] = "Hello World"
std::cout << string;

//Вывод: Hello World
//но где остальные символы?
```
как только поток вывода `std::cout` *натыкается на нуль терминатор, вывод прерывается*

```c++
char string [] = {'h', 'e', 'l', 'l', 'o'};
std::cout << string;
std::cout << std::strlen(string);

//Вывод: Hello╠╠╠╠╠╠╠╠╠╠╠╠╠╠╠╠╠╠╠
//Вывод: 29
```

так как мы не указали `\0`, то поток вывода не прекратится пока не найдёт символ, который сможет интерпретировать как `\0`

```c++
char string [] = {'h', 'e', 'l', 'l', 'o', '\0'};
std::cout << string;
std::cout << std::strlen(string);

//Вывод: Hello
//Вывод: 5
```
